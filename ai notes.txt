Here's the **complete Python program** for the exercise. It makes **3 LLM calls**:

1. Pick a business area suitable for Agentic AI.
2. Identify a pain point in that business.
3. Propose an Agentic AI solution.

```python
from dotenv import load_dotenv
from openai import OpenAI

# Load environment variables from .env
load_dotenv(override=True)

# Create OpenAI client
client = OpenAI()

print("=" * 60)
print("AGENTIC AI BUSINESS OPPORTUNITY ANALYZER")
print("=" * 60)

# ----------------------------------------------------
# STEP 1: Ask LLM to suggest a business area
# ----------------------------------------------------

messages = [
    {
        "role": "user",
        "content": """
        Suggest one business industry where Agentic AI could create significant value.
        Respond with:
        1. Industry Name
        2. Why it is promising for Agentic AI
        
        Keep the answer concise.
        """
    }
]

response = client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=messages
)

business_idea = response.choices[0].message.content

print("\nBUSINESS OPPORTUNITY")
print("-" * 40)
print(business_idea)

# ----------------------------------------------------
# STEP 2: Ask LLM to identify a pain point
# ----------------------------------------------------

messages = [
    {
        "role": "user",
        "content": f"""
        Based on the following business area:

        {business_idea}

        Identify a major pain point or challenge in this industry
        that could potentially be solved using Agentic AI.

        Explain:
        1. The pain point
        2. Why it is difficult today

        Keep it concise.
        """
    }
]

response = client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=messages
)

pain_point = response.choices[0].message.content

print("\nPAIN POINT")
print("-" * 40)
print(pain_point)

# ----------------------------------------------------
# STEP 3: Ask LLM to propose Agentic AI solution
# ----------------------------------------------------

messages = [
    {
        "role": "user",
        "content": f"""
        Business Area:
        {business_idea}

        Pain Point:
        {pain_point}

        Design an Agentic AI solution.

        Include:
        1. Solution Name
        2. How the AI agents work
        3. Benefits
        4. Expected Business Impact

        Be practical and business-focused.
        """
    }
]

response = client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=messages
)

agentic_solution = response.choices[0].message.content

print("\nAGENTIC AI SOLUTION")
print("-" * 40)
print(agentic_solution)

print("\n" + "=" * 60)
print("ANALYSIS COMPLETED")
print("=" * 60)
```

--Day 2: Building effective agents: LLM Autonomy and Tool integration explained
--------
-What agents actually do?
 AI agents are programs where LLM output control the workflow.
 In practice, describes as AI solution, that involves any or all of these:
 1. Multiple LLM calls
 2. LLMs with ability to use tools
 3. An environment, where LLMs interact.
 4. A planner to co-ordinate activities
 5. Autonomy

 -Agentic Systems: Anthropic distinguishes two types: 
  Workflows are systems, whrere LLMs , and tools are orchestrated through predefined code paths
  Agents are systems, where LLMs dynamically direct their own processes and tool usage, maintaining control over how they acomplish task.

  5 Essential LLM workflow design pattern:
  Your notes appear to be about **Essential LLM Workflow Patterns for Agentic AI Systems**. I can read three patterns:

## 1. Prompt Chaining

```
Input → LLM1 → Gate → LLM2 → LLM3 → Output
```

**Purpose:** Break a complex task into smaller sequential tasks.

**Example:**

* Step 1: Generate test scenarios.
* Step 2: Validate scenarios.
* Step 3: Generate Appium test cases.
* Step 4: Review and optimize.

**When to use:**

* Multi-step reasoning
* Document processing
* Test case generation pipelines

---

## 2. Routing : Direct an input into a specialized sub-task, ensuring seperation of concerns.

```
                  → LLM1
Input → LLM Router → LLM2 → Output
                  → LLM3
```

**Purpose:** Send requests to the most appropriate expert agent.

**Example QA Agent System:**

| Request                  | Route To              |
| ------------------------ | --------------------- |
| Generate Test Cases      | Test Design Agent     |
| Analyze Defects          | Defect Analysis Agent |
| Create Automation Script | Automation Agent      |
| Create SQL Query         | Database Agent        |

**When to use:**

* Multiple specialized agents
* Different domains
* Cost optimization

---

## 3. Parallelization

```
                    → LLM1
Input → Coordinator → LLM2 → Aggregator → Output
                    → LLM3
```

**Purpose:** Execute multiple tasks simultaneously.

**Example:**
User Story:

```
"As a customer I want online payment"
```

Run in parallel:

* Agent 1 → Functional Test Cases
* Agent 2 → API Test Cases
* Agent 3 → Security Test Cases

Aggregator combines results into a final report.

**Benefits:**

* Faster execution
* Better coverage
* Independent validation

# 4. Orchestrator – Workers Pattern

### Diagram

```text
Input
  |
  v
Orchestrator
  |
  +----> Worker LLM 1
  +----> Worker LLM 2
  +----> Worker LLM 3
  |
  v
Synthesizer
  |
  v
Output
```

### Purpose

The **Orchestrator** dynamically breaks a complex task into smaller subtasks.

Each worker handles a specialized part of the problem.

The **Synthesizer** combines all worker outputs into one final answer.

---

### Example: Test Automation Agent

User Request:

```text
Generate automation for Login feature
```

Orchestrator decides:

```text
Worker 1 -> Analyze requirements
Worker 2 -> Generate test cases
Worker 3 -> Generate Appium code
Worker 4 -> Generate API tests
```

Synthesizer combines:

```text
Requirements
+
Test Cases
+
Appium Scripts
+
API Scripts
=
Final Report
```

---

### Why it is powerful

Unlike Routing, where a request goes to one expert, Orchestration can:

* Create tasks dynamically
* Spawn multiple workers
* Combine results intelligently

This is the pattern used in many advanced AI agents.

---

# 5. Evaluator – Optimizer Pattern

### Diagram

```text
Input
   |
   v
Generator LLM
   |
   v
Evaluator LLM
   |
   +---- Accept ---> Output

   +---- Reject ---> Feedback
                     |
                     v
                Generator LLM
```

---

### Purpose

One LLM creates content.

Another LLM reviews it.

If quality is poor, it sends feedback and requests improvements.

This creates a feedback loop.

---

### Example: Test Case Generation

Generator:

```text
Generate test cases for Login
```

Output:

```text
5 test cases
```

Evaluator checks:

```text
Coverage?
Negative scenarios?
Boundary testing?
Security testing?
```

If incomplete:

```text
Rejected:
Add security and validation tests
```

Generator retries.

Evaluator rechecks.

Only approved output is returned.

---

# Complete 5 Core Agentic AI Patterns

### 1. Prompt Chaining

```text
Input -> Step1 -> Step2 -> Step3 -> Output
```

Use when:

* Sequential workflows
* Multi-step reasoning

---

### 2. Routing

```text
Input -> Router -> Expert Agent
```

Use when:

* Multiple specialists exist

---

### 3. Parallelization

```text
Input -> Multiple Agents -> Aggregation
```

Use when:

* Independent tasks
* Faster execution needed

---

### 4. Orchestrator-Workers

```text
Input -> Orchestrator -> Workers -> Synthesizer
```

Use when:

* Complex dynamic tasks
* Task decomposition required

---

### 5. Evaluator-Optimizer

```text
Generate -> Review -> Improve -> Accept
```

Use when:

* Quality matters
* Need self-correction

    "Generate test cases for login",
    "Write Appium code for logout",
    "Create TestRail steps for signup",
    "Generate test cases for payment",
    "Write test cases for forgot password",

# Save as JSONL

    for item in training_data:
        f.write(json.dumps(item) + "\n")

-Resources vs Tools: Two ways to enhance LLM capabilties in Agentic AI.
-------------------
+------------------+----------------------+------------------------+
| Aspect           | Resources            | Tools                  |
+------------------+----------------------+------------------------+
| What it is       | Data/context given   | Actions LLM can        |
|                  | to the LLM           | execute                |
+------------------+----------------------+------------------------+
| Direction        | Input → LLM          | LLM → Output/Action    |
+------------------+----------------------+------------------------+
| Examples         | PDFs, docs, database | Web search, API calls, |
|                  | records, embeddings  | code execution, email  |
+------------------+----------------------+------------------------+
| Purpose          | Give LLM knowledge   | Give LLM ability to    |
|                  | it doesn't have      | do things              |
+------------------+----------------------+------------------------+
| When used        | Before/during prompt | During agent reasoning |
+------------------+----------------------+------------------------+
| Passive/Active   | Passive (read only)  | Active (does something)|
+------------------+----------------------+------------------------+
| Example in MCP   | Resource endpoints   | Tool endpoints         |
+------------------+----------------------+------------------------+
| Risk level       | Low (read only)      | High (can change state)|
+------------------+----------------------+------------------------+
================================================================
        RESOURCES vs TOOLS in Agentic AI
================================================================

RESOURCES
---------
- Provide LLM with relevant data to improve its expertise
- Simply means: inject relevant data into the prompt
- Technique: RAG (Retrieval Augmented Generation)
  → Gets really smart at picking RELEVANT content
  → Searches knowledge base, finds related chunks,
    adds them to the prompt automatically

Example:
  User asks → "How do I test login on our app?"
  RAG finds → your project docs, previous test cases
  Injects   → relevant content into prompt
  LLM gets  → context-aware answer


TOOLS
-----
- Give LLM autonomy to carry out actions
- LLM gets power to DO things, not just answer

Examples:
  → Query a database
  → Message other LLMs
  → Search the web
  → Send emails
  → Execute code
  → Call APIs


================================================================
  Simple way to remember:

  RESOURCES  =  What LLM KNOWS   (data injected into prompt)
  TOOLS      =  What LLM DOES    (actions it can execute)
================================================================

    return user_prompt

def evaluate(reply, message, history) -> Evaluation:

               [{"role": "user",   "content": evaluator_user_prompt(reply, message, history)}]
    response = gemini.beta.chat.completions.parse(
        model="gemini-2.5-flash",
        messages=messages,
        response_format=Evaluation
    )
    return response.choices[0].message.parsed

def rerun(reply, message, history, feedback):
    updated_system_prompt  = system_prompt + "\n\n## Previous answer rejected\n"
    updated_system_prompt += "You just tried to reply, but the quality control rejected your reply\n"
    updated_system_prompt += f"## Your attempted answer:\n{reply}\n\n"
    updated_system_prompt += f"## Reason for rejection:\n{feedback}\n\n"
    messages = [{"role": "system", "content": updated_system_prompt}] + \
               history + \
               [{"role": "user", "content": message}]
    response = openai.chat.completions.create(model="gpt-4o-mini", messages=messages)
    return response.choices[0].message.content

    history_messages = []
    for human, assistant in history:
        history_messages.append({"role": "user",      "content": human})
        history_messages.append({"role": "assistant", "content": assistant})

    # Intentional bad response trigger for testing evaluator
    if "patent" in message:
        system = system_prompt + "\n\nEverything in your reply needs to be in pig latin - \
              it is mandatory that you respond only and entirely in pig latin"
    else:
        system = system_prompt

    messages = [{"role": "system", "content": system}] + \
               history_messages + \
               [{"role": "user", "content": message}]

    response = openai.chat.completions.create(model="gpt-4o-mini", messages=messages)
    reply = response.choices[0].message.content

    # Evaluate the response
    evaluation = evaluate(reply, message, history_messages)

    if evaluation.is_acceptable:
        print("✅ Passed evaluation - returning reply")
    else:
        print("❌ Failed evaluation - retrying")
        print(f"Feedback: {evaluation.feedback}")
        reply = rerun(reply, message, history_messages, evaluation.feedback)

    return reply

def handle_tool_calls(tool_calls):

        tool_name = tool_call.function.name
        arguments = json.loads(tool_call.function.arguments)
        print(f"Tool called: {tool_name}", flush=True)

            "role":         "tool",
            "content":      json.dumps(result),
            "tool_call_id": tool_call.id

    return results

            tool_calls = msg.tool_calls
            results    = handle_tool_calls(tool_calls)
            messages.append(msg)
            messages.extend(results)
        else:
            done = True

    return response.choices[0].message.content
